## Key Questions
1. 为什么“翠顶夫人”和“黑羽夫人”会被直接强制变成“黑羽夫人”？
   - **理由**：因为两者在 `洛克王国_蛋组精灵表.json` 中的 `family_chain` 被串在了一条线上，导致旧代码中通过 `split(" → ").pop()` 取出的 `maxStageName` 始终是排在最后的 “黑羽夫人”。
2. 如何在不修改 JSON 数据的前提下优雅地解决分支进化的升级冲突？
   - **决策**：在 `petHelper.ts` 中预处理所有精灵的进化链。对于含有分支进化（即同一进化链上有多个最大 `stage` 精灵）的族群：
     - 若当前宠物本身已处于最高 `stage`，其 `maxStageName` 设为自己本身，避免在输入时被升级到同阶的其他分支。
     - 若当前宠物处于低阶 `stage`（如乖乖鹄、蓝珠天鹅），其 `maxStageName` 默认升级为进化链中最后的那个最高阶形态（“黑羽夫人”），但在头像上提供所有同链最高阶的选择。
3. 如何让“翠顶夫人”和“黑羽夫人”可以在头像下拉框中互相切换？
   - **决策**：重构 `getAvailableSprites`。若宠物属于最高阶，且其进化链有多个同阶的最大形态，则将这些最高阶形态及其子形态都作为可用形态合并返回。这样，在头像的形态选择框中便会出现这两个平级的最终进化，用户一键即可在它们之间切换，且系别也能够自适应。
