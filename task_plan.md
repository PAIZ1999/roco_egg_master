## Key Questions
1. 父母本的身高体重应该是什么数据类型？
   → 决定：身高 (height) 和体重 (weight) 都是 string 类型，方便用户自由输入数字或清除，界面上在输入框左侧分别放置尺子 (Lucide Ruler) 和秤砣 (Lucide Weight) 图标，右侧以文字 `m` 和 `kg` 作为单位后缀。
2. 自动配组结果界面该怎么排版？
   → 决定：放置于父本库和母本库的下方，作为一个高亮显示的“🧬 智能配对与导入中心”面板。只有在父本和母本同时勾选时，才会实时计算出所有符合规则 of 配对组合（相同牌子 && 蛋组交集非空），并提供一键导入按钮。
3. 导入时对蛋窝（`EggPet`）的数据补充该怎么做？
   → 决定：新生成的 EggPet 状态默认为“有现蛋”，极限蛋默认为“无极限蛋”，根据父母三围是否完全相同，自动计算 3V/否 状态。
4. 繁育配对如何支持单对和勾选导入？
   → 决定：通过在 `App.tsx` 中使用 `excludedPairKeys` 状态来记录未被勾选的配对，直接点击卡片任意位置可切换勾选状态（左上角以绿勾表示，未勾选时显示空心圆并降低卡片不透明度）。同时把导入按钮做大，并使用 `e.stopPropagation()` 阻止冒泡避免触发卡片勾选切换。

## Decisions Made
- **Tab 切换**: 在 Header 下方引入 `Nest & Trade` 和 `Parents Hub` 两个标签切换。
- **父母本数据存储**: 数据与 data.pets 处于同级，直接存入 `roco_egg_data.json` 下的 `parents` 数组中，确保无需修改 Electron 主进程代码。
- **单个性格选择**: 每个父母本卡片只支持选择一个性格，数据结构字段为 `nature: string`，移除性格的动态增减逻辑。一键导入时，将 `nature` 作为单元素数组传给 `EggPet` 的 `fatherNatures` 或 `motherNatures`  
- **三围选择**: 沿用原有的三围选择（3个小徽章点击弹出隐式 select 菜单）。
- **单对一键导入**: 在繁育配对卡片里为每对父母本加上了“导入”按钮，使用户既能一键导入全部，也能选择性导入某对。
- **卡片勾选控制**: 点击卡片任意位置切换选中状态，顶部一键导入按钮只导入被选中的多组配对。
