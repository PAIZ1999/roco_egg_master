# 项目知识库: 洛克王国孵蛋数据管理系统 (桌面打包与本地数据存盘)

## 📌 项目概述
该项目是洛克王国宠物孵蛋数据管理系统，基于 Vite + React + TailwindCSS 开发。经过 Electron 移植，现已支持打包为双击运行的 Windows 桌面 `.exe`（便携免安装单文件版），并实现将数据自动持久化到本地物理文件夹中的 JSON 文件中。

---

## 🛠 架构设计与技术栈

1. **渲染进程 (Vite + React)**:
   - 包含主界面展示、筛选条、自动保存状态栏。
   - 文件通过 `vite.config.ts` 中的 `base: './'` 使得构建 of 静态资源支持以 `file://` 协议正确读取。
   - **网页端部署静态资源拷贝**：为解决网页端（如 Vercel、Cloudflare Pages）部署后根目录 `images` 文件夹内图片丢失导致图片裂开的问题，新增了 `copy-images.js` 脚本，并在 `package.json` 中的 `build` 命令后追加 `node copy-images.js`，使打包后自动将根目录 `images` 递归拷贝至 `dist/images`。
   - 数据变更（`pets`、水印配置等）会触发自动保存，优先使用 `window.electronAPI.saveData` 保存至本地，同时也保留了 `localStorage` 的备份。
   - **智能 3V 蛋与极限蛋统计检测**:
     - **3V 蛋**: 当父母三围（生命、物攻、速度等）设置完全一致时，系统会自动将该宠物行高亮，并在表头左侧显示“3V蛋”标志。只有当父母三围一样且窝点状态为“有现蛋”时，才会统计入顶部的“3V蛋”总数。当任一三围被设置为“无”或者三围被隐藏时，系统会自动将该行排除在 3V 统计之外。
     - **极限蛋**: 只有当选择为“极限”且窝点状态为“有现蛋”时，才会统计入顶部的“极限蛋”总数.
   - **洛克王国专属 30 性格清洗系统**:
     - **30 种有效加成性格**: 严格遵循《洛克王国：世界》设定，项目去除了所有“平衡”性格（如实干、认真、浮躁），仅保留 30 种有属性加成的有效性格和 1 个“错性格”。
     - **全生命周期智能数据清洗与空性格值处理**: 提供了 `cleanNature` 清洗模块。当从本地磁盘、localStorage 或备份中加载老旧数据时，会自动将旧的简称（如 `逞强`）或仅含加减号的属性（如 `+物攻 -生命`）无缝补全为全称（如 `逞强 (+物攻 -生命)`）。若遇到已废弃的旧版平衡性格或其它无效字符，则自动转换并兜底为 `"错性格"`，确保了数据的绝对纯净和零宝可梦残留。**特别地，在 2026-06-06 的优化中，添加精灵时的父/母性格默认初始化为空字符串 `""` 并显示为“选择性格”占位符，直接点击即可自动弹出推荐性格下拉列表；`cleanNature` 会特殊保留空字符串和“选择性格”，使其不再被误重置为“错性格”。**
   - **三围隐藏与“无”选项**:
     - **“无”选项**: 父母三围列表选项中新增了“无”，用于标记用户不想填写某项三围的情况，方便区分未设定和已设定的状态。
     - **隐藏三围开关**: 每一行表格的三围数据列都带有一个“隐藏三围”按钮，点击后三围选择框隐藏，只在表格内留白并显示“三围已隐藏”字样以及一个“显示并填写”按钮。
   - **列物理布局与长图导出**:
     - 物理排列顺序为 `牌子 -> 窝点详情 -> 是否极限`，且去掉了包裹 table 容器的 `no-scrollbar` 样式。这样在窄屏时可以通过原生滚动条平滑横向滚动，避免元素被圆角卡片裁剪，确保全部列完全可访问。
     - **长图导出深度优化**: 导出长图时会在 `document.body` 动态添加 `exporting` 样式类，结合 `index.css` 全局规则，不仅过滤垃圾桶、排序手柄、筛选框、下拉框小箭头，还可以将隐藏三围状态下的“显示并填写”等一系列页面动作按钮（带有 `.action-buttons` 类）完美隐藏，使用户得到的长图绝对清爽和完美。
     - **牌子选项扩充**: 品牌选项（`brand`）中除原先的“大粗、大婉、单牌、普通”外，新引入了“小粗”与“小婉”两个全新的预设牌子类型，并赋予它们优雅醒目的高饱和度/轻量化渐变底色，以便在看板和长图中进行醒目的视觉标识。
     - **下拉框 Portal 挂载优化**: 针对精灵名称和性格下拉框在表格行和横向滚动容器中容易被裁剪遮挡的问题，将 Autocomplete 组件的下拉菜单通过 React Portal 挂载到 `document.body` 上，并动态实时计算输入框在视口中的位置，实现了滚动和缩放下的自适应绝对定位。
     - **长图导出排版与对齐优化**:
       - 调大了“精灵名称”列的宽度占比（主表格 14%，导出克隆表 15%），避免长图导出时因为列宽不足导致宠物名字折行。
       - 在将 `input`/`select` 转换成静态 `div` 时，自适应提取其 `text-left`/`text-right` 等对齐属性，并映射到 `justifyContent` 与 `textAlign`，确保导出的长图文字完全对齐不跑偏。
       - **DPI与Zoom缩放自适应校正**: 在启用了 Windows 系统缩放（如 125%、150%）或浏览器页面缩放的电脑上，通过 `getComputedStyle` 取得的物理像素值（如 `fontSize`, `lineHeight` 等）会被乘以 DPR 放大。在导出克隆为静态 `div` 时，停止调用物理像素除以 DPR 重写字号的逻辑，而是完美让其继承原有的 Tailwind 文本字号类（如 `text-[10px]`, `text-xs` 等）和自适应排版。同时，对高度、宽度、padding 等外廓和定位像素进行 `adjustPx` 转换。这彻底规避了 Chromium 内核对中文字体“最小 12px 保护限制”引起的字体溢出容器和换行 Bug，使长图在任意电脑缩放比例（1.25x, 1.5x, 2.0x）下均能稳定、高清地输出，完美无折行。
     - **系别 Badge 纯图标化展示**: 删除了属性 Badge 里的系别中文文字，仅以圆形图标（外加 hover title 说明）呈现，确保界面在小屏幕与导出长图中更加紧凑美观。
      - **自建换蛋需求中心卡片蛋组集成与防抖动排版**:
        - 在换蛋卡片（看板）中引入精灵蛋组 Badge 的显示，支持多蛋组渲染，并复用主表格的 `getEggGroupStyle` 函数确保视觉一致性。
        - 针对“错性格”等长性格名情况，通过将性格/牌子容器行与蛋组容器行完全分离为独立的块级垂直布局，解决了文本溢出挤压或遮挡蛋组的问题，保证了卡片在极端文字长度下的高度自适应与导出长图的美观性。
     - **精灵头像悬浮删除按钮**: 移除了原有的普通删除按钮，重构为完全嵌在精灵头像容器（`relative group/avatar`）内的绝对定位遮罩按钮。在 hover 时通过不透明度渐变（`group-hover/avatar:opacity-100`）呈现半透明红色蒙版及垃圾桶图标，并让头像图片微缩（`group-hover/avatar:scale-75`），提供高品质微交互效果。同时该按钮带有 `.action-buttons` 类，可在导出长图时自动过滤。
     - **精灵蛋窝卡片化与易用性精细打磨**:
       - 将原来杂乱的列表重构为卡片化网格，使精灵名显示在头像正下方。
       - 删除了父母性格旁的 `+` 增加性格按钮以保持布局简约；将三围图标的底盘变更为白底带浅色边框与微阴影，以高对比度衬托彩色 PNG 属性图片。
       - **蛋窝卡片排版重构与美化 (2026-06-09)**:
         - 合并状态/牌子/极限：将原来的“牌子”、“极限蛋”与“蛋窝状态”选项合并到同一行，使用 12 列自适应网格（`grid-cols-12 gap-1`）紧凑排版，腾出垂直空间并提高视觉整体感。
         - 蛋组视觉隔离：蛋组配置区域加上了优雅的浅色背景底框（`bg-slate-50/70 p-1.5 rounded-lg border border-slate-100/60`）并带有高对比度的 Teal 彩色 Badge，起到了极佳的视觉分区作用。
         - 调大核心字体：所有核心字段输入和下拉选择均被调大或加粗（下拉选择框调大至 `text-xs font-bold`，性格输入框调大至 `text-[13px] font-semibold`），在屏幕端和 1200px 高清长图导出中均保持极高的清晰度与可读性。
       - **状态选项简化与新旧数据迁移 (2026-06-09)**:
         - 将原本冗长的 `"已撤窝，要提前换产线"` 状态缩减为简明扼要的 `"已撤窝"`，并统一在类型、下拉、样式映射中对齐。
         - 在 `src/App.tsx` 数据加载模块的 `migratePets` 方法中加入历史数据过渡检查，若加载的历史数据中存在 `"已撤窝，要提前换产线"` 时，将自动原地无缝迁移为 `"已撤窝"`，保证用户已有存储的绝对向下兼容性。
       - 主卡片区上方新增“我的精灵蛋窝中心”标题栏，并在长图导出时动态将标题文本替换为“我的窝点”，使长图的上下排版风格达到完全的对称与美观。
       - **小标题一致性与卡片排版细节极致打磨**：
         - 完美拉齐了“我的精灵蛋窝中心”与“自建换蛋需求中心”标题栏的内边距、标题大小（`text-lg font-bold`）、说明小字（`text-xs mt-0.5`）、图标尺寸（`w-5 h-5`）与 Badge 大小。
         - 左侧列将头像、名字输入框、系别图标封装在 `gap-1.5` 的 Flex 容器中，且将名字输入框的底边距替换为对称 `py-0.5`，实现了头像-名字-系别的像素级等距垂直紧凑排版。
         - 右侧列布局从 `justify-between` 变更为 `justify-start gap-1.5`，消除未选择“有现蛋”时元素被强制拉开产生的中间空虚感。
         - 当窝点状态为非“有现蛋”（如“正在孵”、“已撤窝”、“投资中”等）时，在底部自动渲染一行在高度、外边距、内边距上与“现蛋数量输入”完全对齐的占位说明色带（包含自适应背景、边框及呼吸灯指示点），从而在所有状态下保持卡片整体高度恒定且高度饱满，彻底优化了长图导出与屏幕渲染。
        - **换蛋卡片精灵头像放大与自适应**:
          - 将自建换蛋卡片的头像容器由原本的 `w-16 h-16` 扩大为 `w-20 h-20` (80px)，并将内部精灵图片的尺寸调整为 `w-16 h-16`，Egg 占位图标设为 `w-10 h-10`，右下角属性图标调整为 `w-6.5 h-6.5`（内部图片 `w-4.5 h-4.5`），使得精灵主体在视觉上更加瞩目且清晰，整体布局在拉大后完美自适应，长图导出无溢出。

          - 通过在精灵名称中使用下划线后缀表示特定形态（如 `冬羽雀_春天的样子`），底层重构的 `petHelper` 能够剥离出基础精灵名，从而让多形态精灵也能够完美匹配对应的系别和蛋组。
          - **分支进化与同阶升级重构**：针对在同一进化链上存在平行的多个最终形态（例如“乖乖鹄 → 蓝珠天鹅 → 翠顶夫人 → 黑羽夫人”中的翠顶夫人与黑羽夫人均为 Stage 3；“墨鱿士 → 混乱鱿彩 → 秩序鱿墨”中的混乱鱿彩与秩序鱿墨均为 Stage 2），在 `src/petHelper.ts` 中引入了 `chainMaxStageMap` 预处理。若当前宠物本身已是同链最大形态之一（如“翠顶夫人”或“混乱鱿彩”），则禁止同阶升级以保留其本身；若是低阶形态（如“乖乖鹄”），则默认指向该进化链的最后一个最终态（如“黑羽夫人”）。
          - **最高阶平行形态跨宠物切换**：在 `getAvailableSprites` 中，若宠物处于最终阶段且该链有多个平行最高阶，会同时将所有平行的最高阶形态（及各自的下划线子形态）作为可用形态合并返回。使用户在点击头像形态选择下拉框时，能直接跨宠物在“翠顶夫人”与“黑羽夫人”、或者“秩序鱿墨”与“混乱鱿彩”之间任意切换，切换后对应卡片/行系别与属性动态刷新。
          - **形态下拉框选项文字定制**：在所有相关页面（蛋窝卡片、表格行、换蛋发布看板、换蛋需求看板）的头像形态切换下拉框中，将原本显示为“默认”的最终形态选项进行文本精细化转换：若是翠顶夫人，则显示为“翠顶”；若是黑羽夫人，则显示为“黑羽”；若是秩序鱿墨，则显示为“秩序”；若是混乱鱿彩，则显示为“混乱”——使其完美与其最终进化的名字对应。
          - 蛋窝卡片、表格行、换蛋发布表单以及换蛋需求卡片的头像上均集成了毛玻璃微调形态下拉菜单，点击即可实时切换并更新预览。
          - 自动升级进化链时（如在换蛋卡片或主表中选择进化前精灵）会自动保留形态后缀。
          - 所有的形态选择下拉菜单都加上了 `.action-buttons` 类，使长图导出时能够自动隐形，极简清爽。
          - 换蛋卡片的 hover 删除按钮改为了卡片右上角的淡入垃圾桶按钮，解决了原本 hover 覆盖头像时遮挡多形态下拉菜单的交互痛点。

    - **父母本管理中心与智能繁育配对**:
      - **父母本卡片管理**: 新增“父母本管理”Tab，支持分别录入父本与母本精灵卡片。每张父母本卡片限制仅可选择一个性格（`nature: string`）和一组三围属性（非“无”），并且身高和体重输入框分别集成 Lucide Ruler 图标（单位 `m`）与 Lucide Weight 图标（单位 `kg`）。名字和性格选择支持像蛋窝卡片一样的 Autocomplete 模糊搜索，并根据选择的精灵自动从数据包中匹配对应的蛋组。
      - **智能繁育配对机制**: 自动在已勾选的父本与母本之间寻找匹配组合。规则为：
        1. 蛋组交集非空（即双方至少有一个相同的蛋组）。
        2. 牌子完全相同（如大粗、大婉、小粗、小婉、单牌、普通）。
        若满足上述条件，自动在面板下方实时渲染出子代精灵蛋配对卡片。子代的系别与形态由母本决定（多形态精灵如 `冬羽雀_春天的样子` 在生成子代蛋时，会通过 `petHelper` 自动还原为基础名 `冬羽雀`）。
      - **单配对卡片独立导入与全部导入**: 配对面板上不仅支持导入所选配对，而且在每个配对卡片右下角都配有独立的“导入”按钮。点击此按钮将仅将这一对父母的繁育结果（包括父母的性格、三围、牌子、蛋组等详细映射关系）以“有现蛋”的状态和“无极限蛋”的状态导入至蛋窝中心。
      - **卡片勾选控制与大导入按钮优化 (2026-06-09)**:
        - 调大了每个卡片内的“导入”按钮（`px-3.5 py-1.5 text-[11px]`），并阻止事件冒泡（`e.stopPropagation()`），防止与卡片自身的点击选中动作产生冲突。
        - 支持直接点击繁育卡片任意位置进行“选中”状态切换。
        - 选中时，卡片边框高亮（绿边 `border-emerald-300 ring-emerald-300`），左上角绝对定位显示绿色带勾 `Check` 圆圈。
        - 未选中时，卡片半透明度降低（`opacity-60`），左上角显示灰色空心圆圈，以便进行强烈的视觉对比。
        - 顶部的“一键导入”按钮与选中状态联动，更新为“一键导入所选配对 (X 组)”，仅将已选中的繁育组合导入到蛋窝。
      - **3V 蛋自动识别与排除**: 只有在父母双方设置了完全一致且有效的非“无”三围时，导入蛋窝后的子代蛋才会被自动判定为 `"3V"`；若任一方三围设置为“无”，则自动判定为非 3V 蛋，解决了误判和空值统计漏洞。

2. **主进程 (Electron - CommonJS)**:
   - 文件入口设在 `electron/main.cjs`。由于根目录配置了 `"type": "module"`，Electron 的 CommonJS 代码必须显式命名为 `.cjs` 以确保兼容性。
   - 通过 `preload.cjs` 暴露出安全的 IPC 接口：`loadData()`、`saveData(data)`、`getDataPath()`。
   - **智能存储逻辑**:
     - 启动时：主进程尝试在可执行文件（`.exe`）同级目录下创建一个名为 `data` 的子文件夹，并在其中保存或读取 `roco_egg_data.json`。
     - 权限回退：如果在当前目录下没有写权限（例如安装在系统 `C:\Program Files` 目录下），主进程会自动捕获错误，并将保存路径无缝回退到用户的系统应用数据目录 `C:\Users\<Username>\AppData\Roaming\RocoIncubatorTable\data\roco_egg_data.json`。

3. **打包配置 (`package.json`)**:
   - 使用 `electron-builder` 打包。
   - 打包指令：`npm run electron:build`
   - 生成目标：单文件便携绿色版 (`portable`)，生成于 `dist-electron/` 文件夹下。
   - **打包资源包含**: 在 `build.files` 中明确包含了 `"images/**/*"`，将根目录的 `images` 文件夹一同打包进 `app.asar` 根目录，与 `dist` 同级，确保渲染进程使用 `getImagePath` 计算的相对路径 `../images/...` 能够正常加载图片，解决打包后图片裂开的问题。

## 📂 核心文件目录

- `electron/main.cjs` - Electron 主进程逻辑（窗口控制、文件存盘、路径权限探测）

## 💾 数据存盘规则与恢复

1. **自动保存**：在渲染层对 `pets` 列表及水印参数实施深度侦听（带有 `isLoaded` 标志保全，避免启动时覆盖已有数据），在每次数据变更后延时 600ms 完成磁盘写入。
2. **状态指示器**：网页右上角的 Credits 区域旁配有实时绿/黄呼吸灯。
   - 绿色：已保存，同时在下方以等宽字体展示本地文件绝对路径（如：`存储路径: D:\desk\洛克王国孵蛋表\data\roco_egg_data.json`）。
   - 黄色：正在写入中。
